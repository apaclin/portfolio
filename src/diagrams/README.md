# Статичные схемы (Сценарий A)

SVG-схемы, нарисованные в Figma и экспортированные сюда. Компонент
[`Diagram.astro`](../components/diagrams/Diagram.astro) на сборке инлайнит
нужный по языку страницы файл (ноль клиентского JS, ноль сетевых запросов).

## Почему `src/`, а не `public/`

`public/` отдаётся как статика «как есть» — содержимое нельзя инлайнить.
Здесь файлы импортируются на сборке как строки (`import.meta.glob(..., '?raw')`)
и встраиваются прямо в HTML.

## Конвенция именования

```
<name>.<locale>.svg
```

- `<locale>` — `en` или `ru`.
- Пример: `traditional-ap.en.svg`, `traditional-ap.ru.svg`.
- В компоненте указывается только базовое `name` (`traditional-ap`),
  локаль выбирается автоматически.
- Фоллбэк: если нет `<name>.<locale>.svg`, но есть `<name>.en.svg` —
  используется английская версия + build-warning (как общий RU→EN фоллбэк сайта).

## Рекомендации к экспорту из Figma

- Текст оставлять «живым» (`<text>`, не контуры) — это вес и доступность.
- Один артборд = одна схема. Имя схемы озвучивает обёртка (`aria-label`),
  отдельный `<title>` внутри SVG не обязателен.
